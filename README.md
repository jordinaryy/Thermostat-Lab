# Thermostat-Lab
This project focused on developing a smart thermostat protoype that read room temperature and allows the user to adjust a temperature set point, and control heating or cooling behaviors using a state machine. I built this by working with embedded systems that connects software to physical hardware components like sensors, buttons, LEDs, and an LCD display.

#What did you do particularly well?
One area I performed well in was the multiple hardware components I integrated into a single working system. I spent a lot of time figuring out how each piece connects to one another and because of this I was able to understand State machine logic.

#Where could you improve?
An area I know I can improve on is my code. When implementing the directions to make the machine work I found myself not confident enough and a little overwhelmed. Though this challenge helped me understand how to design a functional product and its intended use.

#What tools and/or resources are you adding to your support network?
The websites [web3schools](https://www.w3schools.com/) and https://gpiozero.readthedocs.io/en/stable/ helped me understand the logic going into assignments as well as the final project when I was creating the prototype. 

#What skills from this project will be particularly transferable to other projects and/or course work?
Skills like hardware to software integration, GPIO programming, and state machine design will help with other course work. These skills will help with any future embedded systems, IoT, and software engineering projects.

#How did you make this project maintainable, readable, and adaptable?
To make this project maintainable and readable, I followed good coding practices like adding comments to explain code and used variable names that stood out. The system itself can be adapted with future improvements like Wi-Fi cloud connectivity.
