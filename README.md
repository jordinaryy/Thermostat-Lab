# Thermostat-Lab/Adding a Temperature and Humidity Sensor
-This project focused on developing a smart thermostat protoype that read room temperature and allows the user to adjust a temperature set point, and control heating or cooling behaviors using a state machine. I built this by working with embedded systems that 
connects software to physical hardware components like sensors, buttons, LEDs, and an LCD display.

-While Temperature and Humitdity sensor was a lab setting up for the Thermostat lab, it showed me how to integrate the hardware with the software. Here I integrated an
AHT20 temperature sensor that read the room temperature.

#What did you do particularly well?
-One area I performed well in was the multiple hardware components I integrated into a single working system. I spent a lot of time figuring out how each piece connects to one another and because of this I was able to understand State machine logic for the Thermostat lab.

-For the temperature and Humidity Sensor lab I had to improvise a little. The instructions at the beginning of the class called for a Raspberry Pi4 but I had a 3. So the AHT20 sensor had to have male to male jumper wires for this to work. I plugged all three into separate spots on the GPIO such as, 3v3, SDA1, and SCL1. This allowed me to a have a functioning temperature reader provided by the AHT20.

#Where could you improve?
-An area I know I can improve on is my code. When implementing the directions to make the machine work I found myself not confident enough and a little overwhelmed. Though this challenge helped me understand how to design a functional product and its intended use.

-When wiring the Temperature and Humidity Sensor lab I could have improved on my overall wiring. My wiring was a little messy and not as straight forward. I could benefit by not overlapping other wires making it difficult to get into certain slots.
